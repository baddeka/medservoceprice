# MedServicePrice

Поиск и сравнение цен на медицинские услуги в Казахстане: анализы, приёмы врачей,
диагностика. Сервис собирает прайсы клиник из открытых источников, приводит разные
названия одной услуги к единому виду и показывает, где дешевле.

Сделано на хакатоне (кейс 1), команда 124.
Демо: https://medservoceprice.onrender.com

## Как устроено

Сбор и обработка данных вынесены в отдельные скрипты, а API работает уже с готовой базой:

- `parse.py` собирает прайсы с сайтов клиник в таблицу `raw_listings`
- `normalize.py` чистит цены, сводит названия к справочнику, убирает дубли и пишет в `listings`
- `backend/` — FastAPI: поиск, фильтры, карточки услуг и клиник
- `frontend/` — обычный HTML/CSS/JS без сборки

Хранилище — SQLite, один файл `medprice.db`.

## Стек

- Python + FastAPI
- SQLite
- rapidfuzz для нечёткого сопоставления названий (если не установлен, используется `difflib`)
- ванильный JS на фронте

## Запуск

Нужен Python 3.10+.

```bash
pip install -r requirements.txt

# демо-данные, чтобы приложение сразу работало
python seed_demo.py

uvicorn backend.main:app --reload
```

Открыть http://localhost:8000, документация API — http://localhost:8000/docs.

Чтобы собрать реальные данные с сайтов клиник:

```bash
python parse.py
python normalize.py
```

Запускать из корня проекта — пути к базе считаются относительно скриптов.

## Эндпоинты

| Метод | Путь | Что делает |
|-------|------|-----------|
| GET | `/search` | поиск с фильтрами (город, категория, цена) и сортировкой |
| GET | `/services` | услуги, сгруппированные, с ценой «от» |
| GET | `/service/{id}` | карточка услуги: описание, подготовка, срок, цены по клиникам |
| GET | `/services/autocomplete` | автодополнение |
| GET | `/clinic/{name}` | карточка клиники со списком услуг |
| POST | `/admin/trigger-parse` | перезапуск сбора данных |

По умолчанию `/search` не отдаёт цены старше 30 дней — чтобы не показывать устаревшие
данные как актуальные.

